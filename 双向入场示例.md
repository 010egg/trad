# 🎯 双向入场条件使用指南

## ✨ 新功能：独立的做多/做空入场条件

现在支持为做多和做空设置**不同的入场条件**！

## 📋 基本用法

### 方式1：震荡策略（推荐）

```json
{
  "symbol": "BTCUSDT",
  "interval": "15m",
  "start_date": "2026-02-01",
  "end_date": "2026-03-02",
  "strategy_mode": "bidirectional",

  "long_entry_conditions": [
    {
      "type": "BOLL",
      "op": "touch_lower",
      "period": 20
    }
  ],

  "short_entry_conditions": [
    {
      "type": "BOLL",
      "op": "touch_upper",
      "period": 20
    }
  ],

  "stop_loss_pct": 3.0,
  "take_profit_pct": 5.0,
  "risk_per_trade": 2.0,
  "leverage": 1
}
```

**说明**：
- 📉 价格触及下轨 → 开多单
- 📈 价格触及上轨 → 开空单
- 🎯 价格回到中轨 → 平仓（自动）

---

## 🎨 策略示例

### 示例1：RSI超买超卖

```json
{
  "strategy_mode": "bidirectional",

  "long_entry_conditions": [
    {
      "type": "RSI",
      "period": 14,
      "op": "lt",
      "value": 30
    }
  ],

  "short_entry_conditions": [
    {
      "type": "RSI",
      "period": 14,
      "op": "gt",
      "value": 70
    }
  ]
}
```

**逻辑**：
- RSI < 30（超卖）→ 做多
- RSI > 70（超买）→ 做空

### 示例5：不对称条件（激进做多，保守做空）

```json
{

  "long_entry_conditions": [
    {
      "type": "BOLL",
      "op": "touch_lower",
      "period": 20
    }
  ],

  "short_entry_conditions": [
    {
      "type": "BOLL",
      "op": "touch_upper",
      "period": 20
    },

      "op": "gt",

    }
```

## 🔄 与旧版本的兼容

### 旧版本（仍然支持）

```json
{
  "strategy_mode": "bidirectional",
  "entry_conditions": [...],  // 做多条件
  "exit_conditions": [...]    // 做空条件
}
```

### 新版本（推荐）

```json
{
  "strategy_mode": "bidirectional",
  "long_entry_conditions": [...],   // 明确的做多条件
  "short_entry_conditions": [...]   // 明确的做空条件
}
```

## 🎓 Python 脚本示例

```python
import requests

def run_oscillation_backtest(token):
    headers = {"Authorization": f"Bearer {token}"}

    payload = {
        "symbol": "BTCUSDT",
        "interval": "15m",
        "start_date": "2026-02-01",
        "end_date": "2026-03-02",
        "strategy_mode": "bidirectional",

        # 做多入场：触及下轨
        "long_entry_conditions": [
            {
                "type": "BOLL",
                "op": "touch_lower",
                "period": 20
            }
        ],

        # 做空入场：触及上轨
        "short_entry_conditions": [
            {
                "type": "BOLL",
                "op": "touch_upper",
                "period": 20
            }
        ],

        "stop_loss_pct": 3.0,
        "take_profit_pct": 5.0,
        "risk_per_trade": 2.0,
        "leverage": 1
    }

    response = requests.post(
        "http://localhost:8000/api/v1/backtest/run",
        json=payload,
        headers=headers,
        proxies={"http": None, "https": None}
    )

    return response.json()
```

## 💡 高级技巧

### 1. 多条件组合（AND逻辑）

同一方向的多个条件**必须同时满足**：

```json
{
  "long_entry_conditions": [
    {"type": "BOLL", "op": "touch_lower", "period": 20},
    {"type": "RSI", "op": "lt", "value": 30},
    {"type": "MACD", "op": "cross_above"}
  ]
}
```

所有3个条件都满足时才做多。

### 2. 非对称风险（不同止损止盈）

虽然目前止损止盈对多空统一，但可以通过调整条件的严格程度来实现：

```json
{
  // 激进做多
  "long_entry_conditions": [
    {"type": "BOLL", "op": "touch_lower", "period": 20}
  ],

  // 保守做空（更严格）
  "short_entry_conditions": [
    {"type": "BOLL", "op": "touch_upper", "period": 20},
    {"type": "RSI", "op": "gt", "value": 75}
  ]
}
```

### 3. 不同周期组合

```json
{
  "long_entry_conditions": [
    {"type": "BOLL", "op": "touch_lower", "period": 20},
    {"type": "MA", "fast": 10, "slow": 30, "op": "cross_above"}
  ],

  "short_entry_conditions": [
    {"type": "BOLL", "op": "touch_upper", "period": 20},
    {"type": "MA", "fast": 10, "slow": 30, "op": "cross_below"}
  ]
}
```

布林带判断短期，MA判断中期趋势。

## 🎯 常见问题

### Q: 如果不设置 short_entry_conditions 会怎样？
A: 系统会使用旧版逻辑，用 exit_conditions 作为做空条件。

### Q: 能否只做多不做空？
A: 设置 `strategy_mode: "long_only"` 即可。

### Q: 多个条件是 AND 还是 OR？
A: 目前是 AND（必须同时满足）。OR 逻辑正在开发中。

### Q: 做多做空能用不同的止损吗？
A: 暂不支持，但可以通过调整条件严格程度间接实现。
